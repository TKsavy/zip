# GitHub Pages Evaluation Site

This folder is the public static website. It has no external dependencies.

## Where To Copy The Sampled Files

After running `sample_eval_pairs.py`, you will have an export folder like:

```text
eval_export/
  sampled_eval.json
  sampled_eval.tsv
  sampling_report.json
  audio/
```

From the project root, prepare the website with:

```bash
python3 prepare_github_pages.py \
  --sampled-dir eval_export \
  --site-dir docs \
  --private-dir private_mapping \
  --dataset-order librispeech_pc,vctk,seedtts,L2arctic,esd
```

The script copies:

```text
eval_export/audio/ -> docs/audio/
```

and writes:

```text
docs/data/evaluation.json
docs/evaluator_scores_template.csv
docs/evaluator_scores_template.xlsx
private_mapping/system_mapping_private.tsv
private_mapping/system_mapping_private.json
```

Upload or publish `docs/` with GitHub Pages. Keep `private_mapping/` private because it reveals which real system corresponds to System A, System B, etc.

The page includes sliders, local browser autosave, progress tracking, and a
`Download my scores` button. Evaluators should enter an evaluator ID, complete
the sliders, download the CSV, and send that CSV back to you.

Optional blank score sheet templates are generated by `prepare_github_pages.py`:

```text
docs/evaluator_scores_template.csv
docs/evaluator_scores_template.xlsx
```

## Page Layout

The page shows one table per dataset. Each row is a sampled item, and columns are:

```text
Sample ID | Source | Reference | System A | System B | System C | ...
```

System labels are fixed across the whole evaluation.

For local preview, serve the folder over HTTP:

```bash
python3 -m http.server 8000 --directory docs
```

Then open `http://127.0.0.1:8000`. Opening `index.html` directly may block the JSON data load.
